# Vibe Coding Tester — 使用说明

> 自动化测试技能：探索页面 → 生成用例 → 执行测试 → 输出报告 → AI 自动修复

---

## 一句话触发

直接把页面丢给我就行：

```
帮我测这个页面 http://your-app.com
```

或者带上产品描述：

```
帮我测这个页面 http://your-app.com
这是一个合租公约 App，支持创建圈子、新增公约、记录守约/违约行为、积分排行榜
```

---

## 两种使用模式

### 模式 A：只给 URL（推荐，最省事）

直接发 URL，我会自己探索页面、理解功能、生成用例，然后跟你确认再跑。

```
帮我测这个页面 http://your-app.com
```

### 模式 B：给 URL + 产品方案

如果你想覆盖得更准，可以附上产品描述。我会直接按你描述的功能生成用例，不需要探索。

```
帮我测这个页面 http://your-app.com

产品说明：
- 首页：输入昵称，创建或加入圈子
- 圈子内：新增公约（设置守约/违约分值）
- 记录行为：选成员 + 选公约 + 选守约/违约，提交后积分实时更新
- 积分榜：支持今日/本周/本月/全部筛选
```

---

## 完整流程

```
1. 探索页面（仅无产品方案时）
        ↓
2. 生成测试用例，展示给你确认
        ↓
3. 你确认后（说"没问题"或"开始"）
        ↓
4. Playwright 自动跑测试 + 截图 UI 检查
        ↓
5. 输出测试报告
        ↓
6. 有失败？AI 自动修复代码 → 重新跑 → 循环（最多5轮）
        ↓
7. 全部通过 → 通知你人工介入做最终验收
```

---

## 用例分级

| 级别 | 说明 | 数量 |
|------|------|------|
| P0 | 核心主流程，全部通过才算合格 | 3~8条 |
| P1 | 重要功能（边界、校验、状态变化） | 各模块 2~5条 |
| UI | 视觉自检（白屏/截断/重叠/对比度/布局），自动追加 | 固定 7条 |

---

## 导出测试报告

测试完可以让我导出报告：

```
帮我把测试报告导出到文档
```

---

## 注意事项

**关于截图里的中文乱码**

测试容器没有中文字体，截图中文会显示为 □ 方块。这是**截图渲染问题**，不是页面 Bug——页面在真实浏览器里显示完全正常。我会在报告里注明。

**关于登录/权限**

如果页面需要登录，告诉我账号密码，或者你先手动登录后再告诉我「可以开始了」。

**关于本地 HTML 文件**

如果页面是本地 HTML 文件（Vibe Coding 生成的），发给我文件路径即可：

```
帮我测这个页面 /path/to/your/app.html
```
